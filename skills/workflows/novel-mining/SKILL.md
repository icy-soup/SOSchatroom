# Novel Mining · 小说对话挖掘工作流

> 将原著小说文本转化为结构化对话数据 → 概率统计 → 角色间态度分析 的完整流水线。

## Overview

本 skill 记录从凉宫春日系列小说原文中提取对话、标注说话对象、计算回应概率、分析角色间态度差异的完整技术流程。核心原则：**原文就有答案**——所有概率和态度特征都从原文统计得出，而非主观设定。

## Pipeline 总览

```
原文小说文本 (.txt)
    │
    ▼
[Phase 1] 对话提取与说话人标注
    人工阅读 + Python 结构化
    │
    ▼
[Phase 2] 概率统计引擎
    条件概率矩阵 / 沉默阈值 / 话题触发权重
    │
    ▼
[Phase 3] 对话对象推断
    addressee 字段 (多轮规则推理)
    │
    ▼
[Phase 4] 角色间态度分析
    语言特征提取 / 关系矩阵 / 偏差分析
    │
    ▼
输出: JSON 配置文件 + 关系矩阵
```

## Phase 1: 对话提取与说话人标注

### 方法

人工阅读原始小说文本，逐条提取对话，记录：

```json
{
  "speaker": "凉宫春日",
  "text": "你们圣诞夜跟人有约吗？",
  "vol": 4,
  "ch": "序曲"
}
```

### 标注规则

- **说话人判断**：使用语义理解（Claude 自身能力），不用正则启发式
- **每卷覆盖率**：完整阅读全书，不批处理/跳读
- **角色范围**：SOS团5角色优先（春日/阿虚/长门/朝比奈/古泉），重要配角保留
- **存储格式**：JSON 数组，`speaker`/`text`/`vol`/`ch` 四字段

### 脚本

`scripts/annotation/annotate_vol*.py` — 每卷一个标注脚本，追加到统一 JSON 文件

### 产出

`data/dialogues/novel_dialogues_annotated.json`

## Phase 2: 概率统计引擎

### 2.1 基础统计

| 指标 | 计算方式 |
|------|---------|
| 各角色对话量 | `Counter(speaker)` |
| 各卷分布 | `Counter(vol)` |

### 2.2 条件概率矩阵

```
P(B 回应 | A 说话) = (A 之后 B 回应的次数) / (A 说话总次数)
```

生成 5x5 矩阵：

```
          触发者
       春日   阿虚   长门   朝比奈  古泉
回应者 春日   —     0.47   0.23   0.35   0.38
       阿虚  0.28   —     0.33   0.26   0.22
       ...
```

### 2.3 沉默容忍度

统计角色两次说话之间的间隔轮次，用于聊天室的沉默唤醒阈值：

| 角色 | 中位间隔 | P(>10轮) |
|------|---------|----------|
| 春日 | 0轮 | 2% |
| 阿虚 | 1轮 | 6% |
| 古泉 | 2轮 | 20% |
| 长门 | 1轮 | 20% |
| 朝比奈 | 5轮 | 36% |

### 脚本

`scripts/analysis/compute_novel_probability.py`

### 产出

`data/analysis/novel_probability_result.json`

## Phase 3: 对话对象推断

### 方法

多轮规则推理引擎，纯 Python (无需 NLP 库)：

| 轮次 | 规则 | 置信度 |
|------|------|--------|
| H1 | 文本中含对方名字 → direct_address | **high** |
| H2 | 上条是问句且当前回答 → qa_pair | **medium** |
| H3 | 上条说话者≠当前 → adjacency | **medium** |
| H4 | 含"大家""你们"→ group_address | **low** |
| H5 | 省略号、独白特征 → self_talk | **low** |
| H6 | 回退到该角色最常对话对象 | **low** |

### 输出增强

```json
{
  "speaker": "凉宫春日",
  "text": "阿虚，你给我过来！",
  "vol": 1, "ch": "第一章",
  "addressee": "阿虚",
  "inference_method": "direct_address",
  "inference_confidence": "high"
}
```

### 脚本

`scripts/analysis/addressee_inference.py`

### 产出

`data/dialogues/novel_dialogues_with_addressee.json`

## Phase 4: 角色间态度分析

### 语言特征提取

对每个 (speaker, addressee) 对计算：

- **平均句长** — 字数统计
- **疑问比例** — `？`/`?` 占比
- **感叹比例** — `！`/`!` 占比
- **命令比例** — 命令句式正则匹配
- **礼貌表达** — "请""谢谢""抱歉"等
- **语气词分布** — "吧""吗""呢""啊""啦""哟"等
- **确定性标记** — "绝对" vs "可能" 的比例
- **亲密称呼** — 是否包含昵称
- **句首词** — 前2字符频率
- **代表性对话** — 最接近中位数的5条

### 偏差分析

```
春日→阿虚 的 命令比例 vs 春日基准值:
  +15% (显著高于平均水平)
```

### 关系矩阵

```
春日→阿虚:   命令式/亲近 (命令+15%, 感叹+20%)
春日→古泉:   客气/正式 (敬语+30%, 感叹-20%)
春日→长门:   直接/功能 (长度-50%, 直接提问+30%)
春日→朝比奈: 逗弄/保护 (感叹+25%, 动作指令+40%)
```

### 脚本

`scripts/analysis/addressee_analysis.py`

### 产出

- `data/analysis/addressee_analysis_result.json`
- `data/analysis/addressee_relationship_matrix.json`

## 关键设计决策

1. **原文优先** — 所有概率和态度特征从原文统计，不是主观设定
2. **纯Python标准库** — 不用NLP依赖，`json` + `re` + `collections` 足够
3. **可审计** — 每项统计结果可追溯到原文对话
4. **分步产出** — 每阶段输出独立 JSON，可单独使用或组合
